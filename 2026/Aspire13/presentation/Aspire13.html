Aspire 13 release wave

What's new in Aspire 13

A thematic presentation of the major changes across Aspire 13.0, 13.1, 13.2, and 13.3, grouped by the real product shifts rather than by release version.

Platform and language support CLI and lifecycle control Dashboard and telemetry Deployment and infrastructure MCP and AI tooling
Platform shiftAspire became multi-language, with Python, JavaScript, and TypeScript AppHost support.
Operational controlThe CLI now manages run, stop, describe, export, restore, and destroy flows.
Telemetry surfaceThe dashboard gained export/import, notifications, browser data, and richer visualization.
Deploy anywhereAzure, Kubernetes, Compose, and registries all became more explicit and portable.

Platform and language support

Current Aspire 13 capabilities focus on a multi-language platform with Python and JavaScript as first-class runtimes and near-parity TypeScript AppHost support.

Python

Production-ready Python app model

  • Run Python scripts, modules, and executables from the app model.
  • Uvicorn/FastAPI support, package-manager detection, and Dockerfile generation.
  • Python version detection and certificate trust automation.

Folder: examples\01-python-app-model
Source: Aspire 13

JavaScript + TypeScript

Unified JavaScript publishing and TypeScript AppHost parity

  • First-class JS publish flows for static sites, Node servers, npm-script apps, and Next.js.
  • Support for Bun, Yarn, and pnpm in modern apphost workflows.
  • TypeScript AppHost parity features such as unified withEnvironment and Compose/deployment hooks.

Folder: examples\02-js-ts-parity
Sources: Aspire 13, Aspire 13.3

CLI and developer workflows

The current CLI is an end-to-end lifecycle tool: build and deploy pipelines, background run control, diagnostics, export, docs lookup, and teardown.

Lifecycle

Full app lifecycle from terminal

  • aspire start, stop, ps, describe, doctor, and restore.
  • aspire secret, aspire certs, aspire wait, and aspire export for ops and automation.
  • aspire destroy for clean resource teardown.

Folder: examples\03-cli-lifecycle
Sources: Aspire 13.2, Aspire 13.3

Pipelines

Pipeline and dashboard-first workflows

  • aspire do powers composable build/publish/deploy pipelines.
  • aspire dashboard run enables standalone dashboard usage.
  • aspire docs api and NativeAOT tool distribution strengthen developer ergonomics.

Folder: examples\04-pipelines-dashboard
Sources: Aspire 13, Aspire 13.3

Dashboard and telemetry

The final state is a richer telemetry surface that combines backend and browser data with export/import, notifications, and programmable APIs.

Telemetry

Unified telemetry operations

  • Import/export telemetry bundles, JSON exports, and .env variable export.
  • Telemetry HTTP API for logs, spans, traces, and real-time follow mode.
  • Notification center with structured command results.

Folder: examples\05-telemetry-operations
Source: Aspire 13.2

Observability

Cross-layer observability in one dashboard

  • Browser console logs, network requests, and screenshots alongside server telemetry.
  • GenAI visualizer improvements including tool definitions, evaluations, and richer payload handling.
  • Standalone dashboard mode for any OTLP source.

Folder: examples\06-cross-layer-observability
Sources: Aspire 13.1, Aspire 13.3

Deployment and infrastructure

Deployment now centers on explicit environments, portable runtime behavior, and clean lifecycle control across Kubernetes, Azure, and Compose.

Kubernetes + Compose

Portable deploy and teardown across runtimes

  • Helm-based Kubernetes deployment with Ingress and Gateway API routing resources.
  • Podman support and privileged Docker Compose publishing for advanced workloads.
  • Container tunnel enabled by default for consistent host connectivity.

Folder: examples\06-deploy
Source: Aspire 13.3

Azure + Registry

Explicit cloud resources and registry flows

  • Azure Managed Redis, deployment slots, and clearer role/compute validation.
  • Explicit container registry resources and push steps in pipeline workflows.
  • Build, deploy, and destroy are now symmetric lifecycle operations.

Agent-native Aspire workflows
